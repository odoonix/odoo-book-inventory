# Developer instructions for `odoo-book-development`

This repository contains the source code and content for the Odoo Book Development 
project, which is a comprehensive guide for developers interested in learning about 
Odoo development and programming. The book covers various topics related to Odoo 
development, including installation, configuration, module development, customization, 
and best practices. The content is organized into different sections and chapters, 
providing a structured approach to learning Odoo development.



# Folder structure

- 'source/' folder contains the book content, which is organized into different sections
  and chapters. this is the main focus for writing and editing the book content and can 
  be modified.
- 'extensions/' folder contains custom Sphinx extensions that are used to enhance the 
  functionality of the documentation. These extensions can be developed and modified as 
  needed to add new features or customize the behavior of the documentation.
- 'build/' folder contains the generated documentation files, which are created by
  Sphinx when building the documentation. This folder is read-only and should not be
  modified directly, as it is generated automatically based on the source files and
  extensions.
- '.vscode/' folder contains project configuration and settings.


# Development guidelines

- all comments, helps, documentation, and code should be in English language.
  This ensures that the codebase is accessible and understandable to a wider audience, 
  including developers from different regions and backgrounds. Using English as the 
  primary language for communication and documentation helps to facilitate 
  collaboration and knowledge sharing within the development community.
- the book content should be written in Persian language, as the target audience for the 
  book is Persian-speaking developers. Writing the book content in Persian allows for 
  better communication and understanding of the concepts and instructions presented in 
  the book, making it more accessible and relevant to the intended audience.
- the code examples and snippets provided in the book should be in English language, as 
  this is the standard language for programming and coding. Using English for code 
  examples ensures that they are universally understood and can be easily referenced by 
  developers regardless of their native language. It also helps to maintain consistency 
  and clarity in the code examples, making them more effective for learning and 
  implementation purposes.


# Typography

- The maximum length of lines MUST be 80 characters. This is a common convention in
  programming and documentation to improve readability and maintainability of the code
  and text. Keeping lines within 80 characters allows for easier reading and prevents
  horizontal scrolling, making it more accessible for developers and readers.
- The book content should be written in a clear and concise manner, using simple and 
  straightforward language. This helps to ensure that the information is easily 
  understood and accessible to a wide range of readers, including those who may be
  new to programming or Odoo development. Avoiding unnecessary jargon and complex
  language can enhance the readability and effectiveness of the book content.
- Code examples and snippets should be properly formatted and indented to enhance
  readability. This includes using consistent indentation, spacing, and line breaks to
  make the code easier to read and understand. Proper formatting helps to improve the
  clarity of the code and makes it more accessible to developers who may be new to
  programming or Odoo development. Additionally, providing comments and explanations
  within the code examples can further enhance understanding and help readers grasp
  the concepts being presented.
